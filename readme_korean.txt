pk_dedicated_server
-----------------------

안내문:

    1. "pk_dedicated_server.exe" 파일을 실행하면 로그 파일과 구성 파일과 데이터 저장 폴더가 생성됩니다.
    2. 구성 파일을 설정하기 위해 "pk_dedicated_server.exe"를 닫으십시오.
    3. 서버를 설정하기 위해 "server_config.json" 파일을 텍스트 편집기로 여십시오.
       모든 내용은 ASCII나 UTF-8로 저장해야 합니다!
    4. 설정에서 보거나 변경할 수 있는 사항은 다음과 같습니다:
        a. basicServerData
            i. name: 전용 서버를 나타내는 이름입니다.
            ii. ipv4: 전용 서버의 로컬 네트워크 IP 주소입니다. 0.0.0.0 주소는 호스트 컴퓨터가 모든 로컬 네트워크 트래픽을 수신할 수 있게 합니다. 특정 네트워크 카드나 인터페이스를 사용하도록 서버를 설정하는 경우가 아니라면 이 부분은 변경할 필요가 없습니다.
            iii. port: 전용 서버의 포트입니다. 방화벽/라우터 설정에서 필요하거나 한 대의 컴퓨터에서 여러 서버를 운영(서버마다 별도의 포트 지정 필요)하는 경우가 아니라면 변경하지 않는 것을 권장합니다.
            iv. saveFolderPath: 저장 데이터가 기록되는 곳입니다. 한 대의 컴퓨터에서 여러 서버를 운영하고자 하는 경우 변경이 필요합니다.
        b. universeSize: (Small/Normal/Large) 전용 서버상의 우주의 크기입니다.
        c. hideConsoleWindow: (true/false) 콘솔 창을 닫습니다. 이 옵션을 사용하면 작업 관리자에서 종료해야만 서버를 닫을 수 있습니다.
        d. admins/users/guests
            i. Credentials
                • password: 사용자 그룹별로 전용 서버 접속에 필요한 비밀번호를 정하십시오. 비밀번호와 권한에 따라 세 가지 사용자 그룹(관리자, 사용자, 손님)을 설정할 수 있습니다. 각 플레이어에게 배정하고자 하는 권한의 사용자 그룹의 비밀번호를 주고, 관리자 비밀번호는 기밀로 유지하십시오!
            ii. canLogin: (true/false) 해당 사용자 그룹이 로그인할 수 있는지를 정의합니다.
            iii. canKickPlayers: (true/false) 해당 사용자 그룹이 서버에서 다른 플레이어를 추방할 수 있는지를 정의합니다.
            iv. canModifyWorld: (true/false) 해당 사용자 그룹이 채광하거나 블록/장식물을 지을 수 있는지를 정의합니다.
            v. canInteractWithLockables: (true/false) 특정 사용자 그룹이 상자와 상호작용할 수 있는지를 정의합니다. 상자의 채광을 방지하려면 canModifyWorld와 함께 사용하면 됩니다.

    5. "pk_dedicated_server.exe" 파일을 다시 실행하여 새로운 설정으로 전용 서버를 시작하십시오.
    6. 전용 서버가 라우터를 통하는 경우(대부분 이에 해당), 게임 데이터가 호스트 컴퓨터에 전송되도록 하기 위해 라우터의 포트포워드 설정을 편집해야 합니다. 이는 라우터의 관리 페이지의 포트포워드 설정 부분에 들어가면 할 수 있습니다. 가능한 경우, 여기서 새로운 서비스(혹은 고객 서비스)를 추가할 수도 있습니다. 새 규칙을 정할 때, 이름을 입력하라고 할 것입니다. 이름은 마음대로 정해도 되지만, 편의상 "portalknights"로 사용하는 것을 권장합니다. 프로토콜은 "UDP"를 선택하십시오. 모든 (내부 및 외부) 포트 번호는 구성 파일에서 변경한 경우(해당 시 구성 파일에 입력한 포트 번호 사용)가 아니라면 "16365"를 사용하십시오. "출력 IP"나 "서버 IP" 부분에는 명령 프롬프트에서 ipconfig을 실행하여 로컬 네트워크 주소를 찾아 입력하십시오. 기타 다른 정보를 요청하는 경우, 그냥 무시하고 진행하십시오. 이 작업을 수행하는 중에 문제가 생기면 https://portforward.com/을 방문하여 참고하십시오. 게임을 호스팅하는 컴퓨터에 고정 IP를 부여하지 않으면 로컬 네트워크 IP 주소가 바뀔 수 있음(따라서 포트포워드 규칙도 무효화됨)을 유의하십시오. 컴퓨터의 고정 IP 주소를 부여하는 방법은 사용하는 라우터의 설명서를 참조하십시오.

    7. 서버에 접속을 허용하고자 하는 사람들에게 외부 IP 주소와 사용자 그룹 비밀번호를 보내십시오. 포트포워드용으로 입력한 IP주소는 로컬 네트워크 IP 주소이지 외부 IP 주소가 아님을 유의하십시오. 사용자들이 접속하는 데 필요한 외부 IP 주소를 확인하려면 http://whatismyip.host/ 등의 웹사이트에 방문해야 합니다. 반드시 사용자들에게IP v6이 아니라 (xxxx:xxxx:xxxx:xxxx:xxxx:xxxx:xxxx:xxxx 의 형태가 아니라 xxx.xxx.xxx.xxx의 형태를 취하는) IP v4 주소를 보내십시오.

    한 대의 컴퓨터에 여러 서버를 설정하려면, 아래의 명령 프롬프트 인수를 사용하여 서버별로 다른 구성을 준비하고 그에 따라 구성 파일을 변경하십시오.
        
    인수:
        a. -config myConfig.json: myConfig.json 구성 파일을 사용하여 서버를 설정합니다.

                      i. 이 이름을 가진 파일이 없는 경우, 이 이름을 가진 기본 구성 파일이 생성됩니다. 참고: 구성 파일의 이름은 임의대로 정할 수 있으며, 여기 사용된 "myConfig.json"은 예시일 뿐입니다.

        b. -log mylogfile.log: 지정된 파일에 로그 파일이 기록됩니다.

            예시:
        pk_dedicated_server -config myOwnConfig.json -log testServer.log
        를 사용하면 myOwnConfig.json 파일에서 구성을 읽어들이며 로그 출력을 testServer.log 파일에 기록하는 전용 서버를 시작합니다.

라이선스:
    개인 또는 독립 서버의 사용

     본 소프트웨어 제품을 구매하고 사용함으로써, 귀하는 본 라이선스 계약의 조항에 따라 소프트웨어 제품을 사용할 수 있는 라이선스 하가만을 취득하였으며, 이에는 개인 또는 독립 서버(이하 “개인 서버”에서 제품의 실행 파일을 배포할 허가가 포함되되, 개인 서버에서 소프트웨어 제품을 사용하고자 하는 모든 제삼자 또한 소프트웨어 제품의 정식 버전의 실행 파일을 포함하며 해당 소프트웨어 제품의 라이선스를 별도로 제공하는 소프트웨어 제품의 정식 버전을 구매해야 합니다.

    귀하는 귀하가 접속을 허가하는 개인 서버의 컨텐츠를 가지고 제삼자가 무엇을 하든지 이에 대해 라이선스 허여자는 책임을 지지 않음에 동의합니다.

    개인 서버의 사용 및 이러한 개인 서버를 통해 생성되거나 배포되는 모든 컨텐츠는 다음을 포함하나 이에 국한되지 않는 요구 조건에 따라야 합니다:

      • 고유한 내용이어야 하며 기타 제삼자의 지적 재산권을 침해하지 말아야 합니다.
      • 접속지 관할권의 법률을 위반해서는 안 됩니다.
      • 모욕적, 외설적, 모독적, 차별적 자료를 사용해서는 안 됩니다.
      • 허위 또는 기만하는 자료를 사용해서는 안 됩니다.
      • 악의적이거나 남에게 피해를 끼치는 자료나 제삼자 또는 라이선스 허여자의 명예를 훼손하는 자료를 사용해서는 안 됩니다.
      • 광고를 포함할 수 없으며, 게임 내 재화에 대해 현금을 요구할 수 없습니다.
      • 라이선스 허여자가 제공하거나 승인한 서버나 자료인 것처럼 행세해서는 안 됩니다.

    귀하가 생성하였거나 제삼자가 귀하의 개인 서버에서 접할 수 있게 하는 컨텐츠를 포함하여 개인 서버에서 접할 수 있는 모든 컨텐츠에 대한 책임은 귀하에게 있음을 분명히 합니다. 이에 따라 귀하는 라이선스 허여자에게 귀하가 소프트웨어 제품을 개인 서버에 호스트할 자격이 있으며, 개인 서버 운영에 있어 본 계약의 조항을 준수할 것이며, 개인 서버의 사용에 있어서 상기에 명시된 대로의 권한이나 라이선스를 라이선스 허여자와 기타 제삼자에게 부여할 권리가 있음을 진술하며 보장합니다.
